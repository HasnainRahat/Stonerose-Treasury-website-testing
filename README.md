# 🧪 Treasury Website Manual Testing Project

📌 Project Overview

This repository contains the complete **manual testing documentation** for the Treasury web application:
🔗 https://stonerosetreasury-iw8u.vercel.app

The goal of this project was to ensure the application's **functionality, usability, and reliability** through structured manual testing. The testing process covered major modules of the system, along with detailed bug reporting.


🎯 Objectives

* Validate core functionalities of the Treasury system
* Identify UI/UX inconsistencies and functional defects
* Ensure smooth user flows across different modules
* Document test cases and defects in a structured format


🧩 Modules Covered

Manual testing was performed on the following modules:

1. **Members**
2. **Sign In**
3. **Sign Up**
4. **Tenant Admin**
5. **Dashboard**
6. **Invites**
7. **Blog**
8. **Account Recovery**
9. **Change Password**
10. **Forgot Password**
11. **Manage Your Account**


📊 Test Artifacts

The repository includes:

* 📁 **10 Excel Sheets**
  Each sheet contains:

  * Test scenarios
  * Test steps
  * Expected results
  * Actual results
  * Status (Pass/Fail)

* 🐞 **Bug Report Document**

  * Total Bugs Identified: **42**
  * Includes:

    * Bug Title
    * Description
    * Steps to Reproduce
    * Expected vs Actual Results
    * Severity/Priority


🔍 Testing Approach

The following manual testing techniques were applied:

* Functional Testing
* UI/UX Testing
* Boundary Value Analysis
* Negative Testing
* Exploratory Testing
* Edge Case Validation


⚙️ Tools Used

* **Microsoft Excel** – Test case design & execution
* **Browser DevTools** – UI inspection & debugging
* **Manual Testing Techniques** – For validation


🚨 Key Findings

* Identified **42 bugs** across multiple modules
* Issues included:

  * Broken validations
  * UI misalignment
  * Navigation inconsistencies
  * Functional failures in forms and workflows


📈 Outcome

* Improved understanding of real-world testing workflows
* Hands-on experience in:

  * Writing structured test cases
  * Identifying critical bugs
  * Reporting defects professionally
